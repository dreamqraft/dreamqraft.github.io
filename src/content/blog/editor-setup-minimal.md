에디터 확장을 정리했습니다. 설치 목록이 늘어날수록 시작이 느려지고, 어떤 기능이 어디서 오는지 알 수 없게 되기 때문입니다. 기준은 하나였습니다 — **지난 한 달간 실제로 썼는가.**

## 남긴 것

| 확장 | 남긴 이유 |
|------|-----------|
| 언어 지원 (Astro, Python) | 문법 강조와 자동완성의 기본 |
| Prettier | 포맷 논쟁을 도구에 위임 |
| GitLens | blame과 히스토리를 에디터 안에서 |
| Error Lens | 에러를 줄 옆에 바로 표시 |

## 걷어낸 것과 이유

- **테마 5종** — 결국 하나만 씁니다.
- **AI 자동완성 중복** — 같은 역할 도구는 하나만.
- **거의 안 여는 파일형식 지원** — 필요할 때 다시 설치하면 됩니다.

## 설정 동기화

설정은 dotfiles 저장소에 `settings.json`으로 관리합니다. 확장 목록도 명령으로 뽑아 함께 커밋해둡니다.

```bash
code --list-extensions > vscode-extensions.txt
```

## 느낀 점

도구를 줄이자 도구를 관리하는 시간도 줄었습니다. 에디터 세팅의 목적은 화려함이 아니라 **마찰 제거**라는 걸 다시 확인했습니다.
