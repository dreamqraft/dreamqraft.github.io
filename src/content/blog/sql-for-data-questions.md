SQL 문법을 아는 것과 **질문을 쿼리로 번역**하는 건 다른 능력입니다. 후자를 연습한 기록입니다.

## 번역의 순서

"지난달 가장 많이 팔린 상품 5개는?"이라는 질문을 받으면:

1. **대상** — 무엇을 세는가? → 주문 행
2. **조건** — 언제/무엇으로 거르는가? → 지난달
3. **묶음** — 무엇 단위로 합치는가? → 상품별
4. **정렬과 개수** — 어떤 순서로 몇 개? → 수량 내림차순 5개

## 순서대로 조립

```sql
SELECT
  product_id,
  SUM(quantity) AS total_qty
FROM orders
WHERE ordered_at >= '2026-04-01'
  AND ordered_at <  '2026-05-01'
GROUP BY product_id
ORDER BY total_qty DESC
LIMIT 5;
```

## 자주 하던 실수

- **날짜를 `BETWEEN`으로 처리** — 월말 마지막 날의 시각 처리가 애매해집니다. `>= 시작, < 다음 달 시작`이 안전합니다.
- **집계 전에 JOIN으로 행 불리기** — 조인 때문에 행이 복제된 상태로 SUM하면 값이 부풀어요. 집계를 먼저 하고 조인하는 걸 기본으로 합니다.

## 연습 방법

실무 데이터가 없어도 공개 데이터셋으로 충분합니다. 핵심은 문제집이 아니라, **일상에서 떠오른 질문을 그때그때 쿼리로 적어보는 것**이었습니다.
