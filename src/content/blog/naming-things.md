"컴퓨터 과학에서 어려운 것 두 가지는 캐시 무효화와 이름 짓기다"라는 농담이 있습니다. 이름이 안 떠오를 때 스스로에게 던지는 질문 세 개를 정리합니다.

## 1. 이 값은 '무엇'인가, '어떻게 만들었나'인가

```ts
// 어떻게 만들었는지를 담은 이름
const filteredMappedPosts = posts.filter(...).map(...);

// 무엇인지를 담은 이름
const recentTitles = posts.filter(...).map(...);
```

만드는 과정은 코드가 이미 보여줍니다. 이름은 **결과가 무엇인지**를 말해야 합니다.

## 2. 이름에 'and'가 들어가는가

`validateAndSave()` 같은 이름이 나오면 함수가 두 가지 일을 한다는 뜻입니다. 이름이 문제가 아니라 함수를 쪼개야 하는 경우가 대부분입니다. **이름 짓기가 어려운 건 종종 설계가 보내는 신호**입니다.

## 3. 반대말이 있는가

- `open` ↔ `close`, `add` ↔ `remove`, `enable` ↔ `disable`

쌍이 되는 동작에 짝이 맞지 않는 단어(`open` ↔ `destroy`)를 쓰면 읽는 사람의 예측이 깨집니다. 프로젝트 안에서 동사 어휘를 통일해두면 이름 짓기 자체가 빨라집니다.

## 마무리

좋은 이름의 기준은 "6개월 뒤의 내가 검색 없이 이해하는가"입니다. 지금 3분 고민한 이름이 미래의 30분을 아껴줍니다.
